# PROMPT PARA AGENTE DE DESARROLLO: CORRECCIÓN Y REESTRUCTURACIÓN DEL MÓDULO DE FORMACIÓN

## CONTEXTO GENERAL

Estoy trabajando sobre una aplicación web que tiene una parte pública y una parte privada.

La aplicación debe quedar simplificada y centrada solo en tres bloques principales:

1. Blog.
2. Formación.
3. Chat asociado a formación.

Este documento se centra especialmente en corregir y rediseñar el módulo de Formación, porque actualmente hay un problema grave en la parte privada al intentar guardar o publicar temas/contenidos.

---

# OBJETIVO PRINCIPAL DE ESTA FASE

Quiero que corrijas, rediseñes y pruebes completamente el módulo de Formación.

Hay dos grandes objetivos:

1. Corregir el error actual de guardado/publicación de temas de formación.
2. Reestructurar el modelo de formación para que funcione por bloques, temas/puntos y contenidos.

La estructura deseada es:

```text
Formación
└── Bloque 1
    ├── Tema/Punto 1
    │   └── Contenido: texto, imágenes, vídeos, adjuntos
    ├── Tema/Punto 2
    │   └── Contenido
    └── Tema/Punto 3
        └── Contenido

└── Bloque 2
    ├── Tema/Punto 1
    ├── Tema/Punto 2
    └── Tema/Punto 3
```

No quiero que los temas estén todos juntos en una lista plana. Tienen que organizarse por bloques.

---

# ERROR ACTUAL QUE HAY QUE CORREGIR

Actualmente, en la parte privada de Formación ocurre lo siguiente:

1. Entro en Formación.
2. Subo o creo un tema.
3. Relleno información.
4. Pulso en “Guardar” o “Publicar”.
5. No ocurre nada.
6. No se guarda nada.
7. No aparece nada en consola del navegador.
8. No aparece ningún error visible.
9. No parece ejecutarse ninguna función.
10. El único cambio visual es que se ilumina o queda enfocado el apartado de descripción.
11. No se lanza ninguna petición visible.
12. No se crea contenido.
13. No se persiste nada.
14. La funcionalidad está bloqueada.

Este error debe tratarse como prioridad absoluta.

---

# DESCRIPCIÓN DEL ERROR PARA CONTEXTO

El problema parece estar en el evento del botón o en el formulario.

Posibles causas a investigar:

- El botón “Guardar” o “Publicar” no tiene `onClick`.
- El botón está dentro de un formulario pero no dispara `onSubmit`.
- El botón tiene `type="button"` cuando debería disparar una acción.
- El botón tiene `type="submit"` pero el formulario no tiene manejador.
- Hay validación silenciosa que bloquea el guardado.
- El foco se mueve al campo descripción porque hay un campo requerido o validación HTML nativa.
- Hay un `preventDefault()` mal colocado.
- La función de guardado no está conectada.
- La función existe pero no se llama.
- La función se llama pero no hace petición.
- La petición no aparece porque hay una condición previa que la bloquea.
- Hay campos requeridos ocultos o no cumplidos.
- Hay un estado React que no cambia.
- Hay un error capturado silenciosamente sin `console.error`.
- El componente no está recibiendo correctamente props o contexto.
- La ruta/API de guardado de formación no existe o no está conectada.
- El botón “Publicar” y “Guardar” no están diferenciados correctamente.
- El estado de publicación no se está enviando.
- El backend no recibe nada.
- El frontend no está mostrando feedback de carga, éxito o error.

No asumas una causa sin revisar el código.

# NUEVA ESTRUCTURA FUNCIONAL DE FORMACIÓN

Quiero que el módulo de Formación se reestructure para trabajar de forma cómoda con esta jerarquía:

```text
Formación
└── Bloques
    └── Temas/Puntos
        └── Contenido
```

## Formación

Cada formación debe tener:

- Título.
- Descripción.
- Imagen principal opcional.
- Estado: borrador/publicada.
- Orden si hay varias formaciones.
- Fecha de creación.
- Fecha de actualización.
- Bloques asociados.

Ejemplo:

```text
Formación: Modelos predictivos en investigación clínica
Descripción: Curso introductorio sobre modelos predictivos, validación y aplicación clínica.
Imagen principal: opcional
```

## Bloques

Cada formación puede tener varios bloques.

Ejemplo:

```text
Bloque 1: Fundamentos de los modelos predictivos
Bloque 2: Desarrollo del modelo
Bloque 3: Validación interna y externa
Bloque 4: Aplicación clínica e interpretación
```

Cada bloque debe tener:

- Título.
- Descripción breve opcional.
- Orden.
- Estado.
- Temas/puntos asociados.

## Temas o puntos dentro de cada bloque

Cada bloque puede tener varios temas o puntos.

Ejemplo:

```text
Bloque 1: Fundamentos de los modelos predictivos

1. Qué es un modelo predictivo
2. Diferencia entre causalidad y predicción
3. Variables candidatas
4. Riesgo de sobreajuste
5. Medidas de rendimiento
```

Cada tema/punto debe tener:

- Título.
- Descripción breve opcional.
- Orden.
- Contenido principal.
- Imágenes opcionales.
- Vídeos opcionales.
- Adjuntos opcionales.
- Estado: borrador/publicado.

# DOS FORMAS DE TRABAJO QUE DEBEN SER POSIBLES

El sistema debe permitir dos formas de creación de cursos.

## Forma 1: Crear primero la estructura y después rellenar contenidos

El administrador puede:

1. Crear una formación.
2. Añadir título, descripción e imagen.
3. Crear todos los bloques.
4. Dentro de cada bloque, crear todos los temas/puntos.
5. Guardar la estructura vacía o semivacía.
6. Volver después.
7. Rellenar los contenidos de cada tema.

Esta forma es útil cuando primero se diseña el índice completo.

## Forma 2: Crear estructura y contenido progresivamente

El administrador puede:

1. Crear una formación.
2. Crear bloque 1.
3. Crear tema 1 dentro del bloque 1.
4. Añadir contenido completo al tema 1.
5. Crear tema 2.
6. Añadir contenido al tema 2.
7. Crear bloque 2.
8. Continuar progresivamente.

Esta forma es útil cuando se va construyendo el curso poco a poco.

Ambas formas deben funcionar.
