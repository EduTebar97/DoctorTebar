1. Rediseñar tarjetas públicas de blogs y formaciones
Problema actual

En la parte pública, las tarjetas de blogs y formaciones muestran descripciones demasiado largas directamente dentro de la tarjeta.

Esto provoca que:

Las tarjetas sean muy grandes.
Haya demasiado texto visible.
El diseño se vea sobrecargado.
La navegación sea poco visual.
Los blogs relacionados ocupen demasiado espacio.
La experiencia no sea limpia ni profesional.
Objetivo

Quiero que las tarjetas sean compactas, visuales y modernas, con un estilo tipo catálogo, biblioteca visual o plataforma tipo Netflix.

Cada tarjeta debe mostrar principalmente:

Imagen opcional, si existe.
Título.
Botón para ver la descripción.
Acceso al contenido al hacer clic en la tarjeta.

La descripción no debe aparecer directamente visible dentro de la tarjeta.

Estructura esperada de la tarjeta
┌─────────────────────────────┐
│ Imagen opcional             │
├─────────────────────────────┤
│ Título del blog/formación    │
│ [Ver descripción]           │
└─────────────────────────────┘
Botón “Ver descripción”

Cada tarjeta debe incluir un botón:

Ver descripción

Al hacer clic en ese botón:

Debe abrirse una ventana modal.
La ventana debe mostrar la descripción completa.
La descripción debe verse con letra grande, clara y legible.
El diseño debe ser elegante y profesional.
Debe aparecer una X arriba a la derecha para cerrar.
El fondo puede oscurecerse suavemente.
Al cerrar, el usuario debe volver al listado sin perder el contexto.
Importante: separar clic de tarjeta y clic del botón

Debe haber dos acciones distintas:

Clic en la tarjeta o en el título
Abre el blog o la formación.
Clic en “Ver descripción”
Abre solo el modal de descripción.

El botón de descripción no debe abrir accidentalmente el blog o la formación.

Usar event.stopPropagation() o una solución equivalente si es necesario.

Aplicar este cambio en

Este cambio debe aplicarse en:

Listado público de blogs.
Listado público de formaciones.
Blogs relacionados al final de cada blog.
Formaciones relacionadas, si existen.
Cualquier zona pública donde aparezcan tarjetas con título + descripción larga.
2. Arreglar el visor ampliado de imágenes
Problema actual

Ya existe una funcionalidad para ampliar imágenes al hacer clic, pero está rota.

Actualmente:

Se abre una ventana/modal.
Aparece la X de cierre.
Pero la imagen no se ve.
La pantalla queda negra, blanca o vacía.
El visor se abre sin renderizar correctamente la imagen.
Objetivo

Quiero que el visor de imágenes funcione correctamente.

Al hacer clic en una imagen:

Debe abrirse un modal.
La imagen debe verse correctamente.
Debe respetar su proporción.
Debe adaptarse al tamaño de la pantalla.
Debe tener una X visible arriba a la derecha.
Debe cerrarse con la X.
Idealmente debe cerrarse también haciendo clic fuera de la imagen o pulsando Escape.
Revisión técnica obligatoria

Revisar especialmente:

Que la URL o src de la imagen se pasa correctamente al modal.
Que el estado de imagen seleccionada no está vacío.
Que el modal no se renderiza sin imagen válida.
Que no hay conflicto entre rutas relativas, base64, blobs, localStorage o archivos subidos.
Que el CSS no oculta la imagen.
Que no hay display: none, opacity: 0, z-index incorrecto o tamaños imposibles.
Que el modal tiene un z-index superior al resto.
Que la imagen usa estilos seguros:
max-width: 90vw;
max-height: 90vh;
object-fit: contain;
Comportamiento esperado
Usuario hace clic en miniatura
→ se guarda la imagen seleccionada
→ se abre modal
→ se renderiza imagen ampliada
→ usuario puede cerrar con X
→ vuelve al contenido anterior

No debe abrirse nunca un modal vacío.

Aplicar en

Debe funcionar en:

Imágenes dentro de blogs.
Imágenes dentro de formaciones.
Imágenes de temas.
Imágenes de recursos.
Miniaturas de tarjetas, si aplica.

El área central muestra la edición de ese tema.

10. Mejorar pegado de texto desde Word en el editor
Problema actual

Cuando escribo un contenido en Word y luego lo copio y lo pego en el editor del blog o formación:

No se respetan bien los párrafos.
Se pierden separaciones.
Se pierden saltos de línea.
Se pierde estructura visual.
Tengo que volver a separar manualmente muchas partes.
Esto me hace perder tiempo.
Objetivo

Quiero que el editor respete lo máximo posible la estructura del texto pegado desde Word.

Comportamiento esperado

Cuando pego contenido desde Word:

Deben respetarse los párrafos.
Deben mantenerse saltos de línea razonables.
Deben conservarse listas si existen.
Deben conservarse títulos o subtítulos si el editor lo permite.
Deben evitarse pegados en un único bloque de texto sin separación.
Debe limpiarse el HTML innecesario de Word para no ensuciar el contenido.
Implementación sugerida

Analizar cuál es el editor actual y aplicar la solución más adecuada.

Posibles soluciones:

Capturar el evento paste.
Leer event.clipboardData.
Detectar si viene HTML desde Word.
Convertirlo a HTML limpio o Markdown limpio.
Mantener párrafos y listas.
Eliminar estilos basura de Word.
Insertar el contenido limpio en el editor.
Requisitos importantes

No quiero copiar todos los estilos visuales de Word.

Quiero conservar principalmente:

Párrafos.
Saltos de línea.
Listas.
Encabezados si se detectan.
Negritas y cursivas si el editor lo soporta.

Pero quiero evitar:

Estilos raros de Word.
Fuentes incrustadas.
Colores innecesarios.
HTML sucio.
Tablas rotas, salvo que el editor las soporte.
Resultado esperado

Si en Word tengo:

Título

Párrafo uno.

Párrafo dos.

- Punto uno
- Punto dos

Al pegarlo en el editor debe quedar aproximadamente:

Título

Párrafo uno.

Párrafo dos.

- Punto uno
- Punto dos

No debe quedar todo pegado en una sola línea.
